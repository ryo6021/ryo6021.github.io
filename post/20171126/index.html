楽に書きたい

会社で書類を作るときは基本的に Word か Excel を使っています。
しかし、見た目の統一感を出すのに手間がかかることが多いと感じます。
軽量マークアップ言語を使って楽に書きたいと考えていました。
このような場合、真っ先に候補に挙がるのが Markdown です。
しかし、 Markdown は表などの表現を不得手としています。
そこで、表現力の高い AsciiDoc を使用することにしました。

また、環境構築の手間を省くために、
標準で AsciiDoc を使える Hugo を利用することにしました。

  • AsciiDoc と PlantUML と mermaid.js で素敵なテキストベース仕様書ライフ

  • Hugo + GitHub Pages でブログを作る #1

自動検査したい

誤字や見た目の統一感を、自力で維持することは困難です。
そこで、マークアップ言語に対応したドキュメント検査ツールとして
RedPen を使うことにしました。
Atom の linter-redpen で書きながら運用しています。
